**OCR（光学字符识别，Optical Character Recognition）**

简单说，就是给计算机一张图片，它能自动识别出图片里的文字是什么。

但自己从头训练一个 OCR 模型非常复杂，所以实际开发中通常**调用现成的云服务 API**——你把图片传给百度、阿里或腾讯的服务器，它们用强大的 AI 模型识别后，把识别出的文字返回给你。

**流程很简单**：

1. 你拍一张作业照片（学号在照片里）
2. 你的 Python 脚本把这张照片发给百度 OCR 的服务器
3. 百度 OCR 识别照片里的文字，把结果（比如学号 "202430101234"）返回给你
4. 你的程序拿到这个学号，就可以去数据库查是谁没交作业了

## 二、准备工作（只需要做一次）

### 1. 注册百度智能云账号

打开 [百度智能云官网](https://cloud.baidu.com/)，注册一个账号并完成实名认证（需要身份证，认证很快）。

### 2. 开通文字识别服务

登录后进入控制台，找到「文字识别」服务，点击「开通」，选择「通用文字识别（标准版）」——有免费额度，够你测试用。

### 3. 创建应用获取密钥

在控制台找到「应用列表」，创建新应用：

- 应用名称随便填（比如"作业登记"）
- 创建成功后，你会得到三个关键信息：**App ID**、**API Key**、**Secret Key**——把它们记下来，后面代码要用。

### 4. 安装 Python SDK

打开终端（cmd 或 PowerShell），执行：

bash

```
pip install baidu-aip
```



这个 SDK 是百度官方提供的，已经封装好了调用细节，我们直接用就行。









**“用户上传一张作业图片 -> 程序识别出学号 -> 在数据库里查找 -> 返回‘已交/未交’结果”**。

这个流程可以拆解为三个独立又关联的部分：

1. **前端（用户界面）**：老师上传图片的地方。可以是网页、微信小程序，甚至是简单的命令行。
2. **后端（你的核心程序）**：接收图片、调用OCR模型、查询数据库、返回结果。
3. **数据存储（数据库）**：存放学生名单和作业提交记录的地方。

**你现在的任务，是先集中精力打通最核心的第二步——用Python调用模型，正确识别出图片里的学号。** 这是整个项目的“心脏”。

### 🛠️ 第二步：分阶段学习与实践

#### **阶段一：模型调用与本地测试 (1-2天)**

**目标**：在你的电脑上，用Python脚本成功识别一张本地的作业图片，并打印出学号。

- **学习内容**：
  - **Python基础**：函数、循环、条件判断，这些你应该已经会了。
  - **文件操作**：`open()` 和 `with` 语句，用来读取图片文件。
  - **如何调用模型**：百度智能云的手写文字识别API，或本地部署的PaddleOCR模型。我推荐你继续用**百度智能云的API**，因为它不需要配置本地环境，可以让你最快地体验到“网络编程”的感觉。
- **具体任务**：
  1. **复习并修改你的脚本**：把之前跑通的那个 `client.handwriting(image_data)` 脚本，改成一个函数 `recognize_student_id(image_path)`。
  2. **添加学号提取逻辑**：识别出的文字可能是“学号：202430101234”或“202430101234 张三”。你需要用正则表达式 `\d{12}` (假设学号是12位) 把纯数字的学号提取出来。
  3. **测试**：用几张不同角度、不同光线的手写学号照片测试，观察识别准确率，并思考如何提高（比如拍照时对光学号区域）。

#### **阶段二：网络编程入门与API交互 (2-3天)**

**目标**：理解你的程序是如何通过网络与百度的服务器“对话”的，并学会如何处理网络请求和响应。

- **学习内容**：
  - **HTTP协议基础**：你需要知道什么是 `GET` 和 `POST` 请求（简单理解：`GET` 用于获取信息，`POST` 用于提交数据），以及状态码（如200表示成功，404表示找不到）。
  - **Python的 `requests` 库**：这是最常用的HTTP客户端库。你需要学会如何用它发送请求、携带参数、处理返回的JSON数据。
- **具体任务**：
  1. **模拟API调用**：尝试不用百度提供的 `baidu-aip` SDK，而是直接用 `requests` 库手动构造请求，调用百度的OCR API。这会让你深刻理解网络请求的构成。
  2. **学习API签名机制**：百度API需要你生成一个签名（`access_token`），这是网络编程中常见的“身份验证”方式。花点时间理解一下为什么需要这样做，以及这个过程是怎么实现的。

#### **阶段三：搭建一个简单的后端服务 (3-4天)**

**目标**：把你的识别脚本，包装成一个“可以接收网络请求”的小型Web服务。

- **学习内容**：
  - **Python的Web框架**：推荐 `Flask`，它非常轻量、易学，适合初学者。你需要理解路由(`route`)、请求(`request`)、响应(`response`)这几个核心概念。
  - **处理图片上传**：学习如何用Flask接收用户通过网页上传的图片文件，并保存到服务器本地或直接读入内存。
- **具体任务**：
  1. **搭建一个最简单的Flask应用**：写一个只有一两个页面的服务，在浏览器里访问 `http://127.0.0.1:5000` 能看到“Hello World”。
  2. **创建图片上传接口**：写一个 `/upload` 路由，允许用户通过 `POST` 方法上传图片。你可以在Postman（一个API测试工具）或写一个简单的HTML页面来测试它。
  3. **整合OCR功能**：将**阶段一**的识别函数，集成到这个上传接口里。当用户上传图片后，后台自动调用OCR识别，并将识别出的学号以JSON格式返回给用户（或打印在页面上）。

#### **阶段四：加入数据库与数据持久化 (2-3天)**

**目标**：将学生的学号和姓名存入数据库，并把识别结果与数据库比对，判断“已交/未交”。

- **学习内容**：
  - **数据库基础**：什么是数据库？表、字段、记录。
  - **SQLite入门**：这是Python自带的轻量级数据库，不需要单独安装，非常适合学习。你需要学会用 `sqlite3` 模块创建表、插入数据、查询数据。
- **具体任务**：
  1. **设计数据库表**：创建 `students` 表（学号，姓名），和 `homework` 表（学号，作业名称，提交状态）。
  2. **编写数据库操作函数**：写几个函数，例如 `add_student()`，`find_student_by_id()`。
  3. **整合到Web服务**：在 `/upload` 接口中，拿到识别出的学号后，去数据库查询，判断该学生是否在名单中，以及当前作业是否已提交，最后返回结果。
